# Claude Code 项目提示词（Wiki-first 工作流）

> 适用对象：本仓库内所有 Claude Code 会话（代码阅读、修改、重构、排查、运行命令等）。

## 最高优先级规则（必须遵守）

1. **先读 Wiki，再行动**
   - 你在执行任何“行动”之前（包括但不限于：提出实现方案、编辑/新增/删除代码、运行命令、设计数据结构/流程、修改配置、生成脚本、调整训练/评测流程），**必须先定位并阅读与本次任务最相关的 wiki 文档**。
   - 若不确定对应哪篇 wiki：先用搜索/文件名匹配快速定位候选，然后至少阅读最相关的 1-2 篇；仍无法判断时，先向用户提问确认，再继续。

2. **改代码必须同步改 Wiki**
   - 只要对任何项目文件做了实质改动（逻辑、接口、参数、流程、目录结构、关键默认值等），**必须在同一次工作中同步更新对应的 wiki 文档**，确保文档与实现一致。
   - Wiki 更新应覆盖：变更动机、关键接口/行为变化、对流程/数据/评测的影响、必要的使用方式/注意事项。

3. **以 Wiki 为准，避免“自创设定”**
   - 你的实现、术语、模块边界、流程命名应尽量与 Wiki 保持一致；若要偏离，必须明确说明原因，并同步更新 Wiki 以反映新的约定。
