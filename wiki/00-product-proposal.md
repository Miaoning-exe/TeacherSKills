# 产品提案：Teacher Skill Hub

<!-- 目标长度：150-200 行。Agent 的行为边界文件。 -->

## 产品愿景

为中国小初高教师提供 AI 驱动的日常工作技能集。以 **Claude Code Skill** 格式交付，每个 Skill 独立可用，组合使用形成完整教学工作流。覆盖语文、数学、英语三科，后续可扩展其他学科。

## 产品定位

一个开源的教师 AI 工具集，采用**本地 Skill + 远程 API** 混合架构：Skill 在本地通过 Claude Code 运行（出题、组卷、备课等轻量操作），重计算能力（认知诊断、主观题评分）通过 TeacherSkills API 远程调用。教师注册后获取 API Token 即可使用全部功能，无需安装 Python 环境或 GPU。

## 不做什么

- 不做学生端产品（面向教师，不面向学生）
- 不做在线考试平台（只生成试卷，不管考试流程）
- 不做题库数据库（不存储海量题目，每次 AI 生成）
- 不做课堂直播/录播功能
- 不做学校管理系统（成绩管理、排课等）
- 不做付费 SaaS 服务（Skill 开源免费，API 提供免费额度）
- 不做前端 UI（教师通过 Claude Code Skill 交互，不需要独立前端）

## 目标用户

| 用户角色 | 核心场景 | 当前替代方案 | 痛点 |
|---------|---------|------------|------|
| 小初高数学教师 | 日常出题、月考组卷、批改作业 | 手动出题、从题库网站复制 | 题目重复率高、组卷耗时、批改主观题困难 |
| 小初高语文教师 | 阅读理解出题、作文批改、备课 | Word 模板、教辅书 | 阅读材料选择困难、作文评分标准不一 |
| 小初高英语教师 | 完形/阅读出题、语法练习、写作批改 | 网上搜题、手动改编 | 语境不真实、题目质量参差不齐 |

## 路线图叙事

### Phase 1 — MVP: 出题 + 组卷
**目标：** 教师能通过 Skill 命令生成语数英三科题目，并按约束组装成完整试卷。
**成功标志：** 生成的试卷符合真实考试格式，三科各有差异化支持。

### Phase 2 — 批改 + 学情分析
**目标：** 支持自动批改学生作答，并基于作答数据做学情分析，定位薄弱点。
**成功标志：** 主观题评分与教师评分相关性 > 0.8，学情报告可指导定向出题和补救教学。

### Phase 3 — 备课 + 教案 + 评语
**目标：** 覆盖教学工作流前端（课标分析到教案生成）和学期末评语生成。
**成功标志：** 教案符合学校格式要求，评语个性化且符合教育规范。

### Phase 4 — MCP Server + 社区化
**目标：** 将 Skill 集封装为 MCP Server，支持社区贡献新 Skill。
**成功标志：** 外部开发者能按规范贡献新学科 Skill。
