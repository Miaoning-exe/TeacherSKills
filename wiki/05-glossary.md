# 术语表

> 统一项目中的教育领域术语，防止 Agent 用错词或混用近义词。

| 术语 | 定义 | 不要混淆 |
|------|------|---------|
| 出题 (chuti) | 根据知识点、题型、难度等参数生成单道题目 | 不等于"组卷"——出题是生成单题，组卷是组装试卷 |
| 组卷 (zujuan) | 按约束条件（难度分布、知识覆盖、总分、时长）将多道题组装为完整试卷 | 不等于"出题"——组卷消费题目，不生成题目 |
| 批改 (gaijuan) | 批改学生作答，给出分数和反馈。客观题本地处理，主观题调用远程 API | 项目统一用"批改"，不叫"改卷""评卷" |
| 学情分析 (xueqing) | 基于批改数据，生成班级/个人知识掌握报告，定位薄弱点并给出补救建议 | 不等于"认知诊断"——学情分析是教师看到的 Skill，认知诊断是背后的远程 API |
| 认知诊断 | 基于学生作答数据，用统计/深度学习模型估计知识点掌握度的算法 | 不是独立 Skill——是远程 API 服务 (`POST /api/diagnosis`)，被学情分析调用 |
| 评语 (pingyu) | 生成个性化的学生期末/阶段性评语 | 不叫"评价"——评价是广义概念，评语特指写给学生/家长的文字 |
| 备课 (beike) | 为某节课做准备：分析课标、梳理知识点、设计教学策略 | 不等于"写教案"——备课是分析和策略，教案是正式文档 |
| 写教案 (jiaoan) | 撰写标准化教案文档：目标/重难点/教学过程/作业/反思 | 不等于"备课"——教案是备课的产出物 |
| 知识点 (KnowledgePoint) | 课程标准中可独立评估的最小知识单元 | 不叫"考点""知识"——统一用"知识点" |
| 题目 (Question) | 一道完整的考核题，含题干、答案、解析 | 不叫"题"或"试题"——代码中统一用 Question |
| 试卷 (ExamPaper) | 由多个题目按分节组装的完整考试文档 | 不叫"卷子""考卷"——代码中统一用 ExamPaper |
| 题型 (QuestionType) | 题目的结构类型：选择、填空、解答等 | 不等于"难度"——题型描述结构，难度描述认知要求 |
| 难度 (DifficultyLevel) | 题目的认知难度：易/中/难 | 不等于"题型"——参见上行 |
| 认知层次 (KnowledgeLevel) | Bloom 分类法的六个层次：识记→理解→应用→分析→评价→创造 | 不等于"难度"——认知层次是教学目标维度，难度是题目参数 |
| 课标 | 国家课程标准，教学内容的权威依据 | 不叫"大纲""教学大纲"——2001 年后统一称"课程标准" |
| IRT | Item Response Theory，项目反应理论，用于单维能力估计 | 不等于 NCDM——IRT 估计单一能力值，NCDM 估计多维知识掌握 |
| NCDM | Neural Cognitive Diagnosis Model，神经认知诊断模型 | 见上行 |
| Skill | Claude Code Skill，一个独立可调用的 AI 工具单元 | 不等于"技能"——本项目中 Skill 特指 Claude Code Skill 格式 |
| API Token | 教师注册后获取的认证凭证，用于调用远程 API | 不叫"密钥""API Key"——项目统一用"API Token" |
